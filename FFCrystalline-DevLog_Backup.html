a while back I had looked into directly modifying the ATB Gauge directly through code, I just figured out how to do it while playing around with debugging. $gameActors._data[1]._tpbChargeTime = 0 (0 is empty and 1 is full, you can also set this to a negative value and it will start even beyond empty and will take time to even reach the starting point on the gauge) $gameActors._data[1]._tpbState = "charging" (if a character has their turn already this is set to "charged", make sure if you manually change the charge time that you set the tpbState back to "charging" or it will not reset it properly) I've only been messing with this for a little while but in practice I've found that it is always best to just set both values every time. I haven't run into any bugs with this yet, and you can also use this to force a character out of their turn even if they are currently in a menu. the menu closes. if you want to make some kind of delay strike during ATB you can also do this: $gameTroop._enemies[0]._tpbChargeTime += -0.4

To solve this I created new item effect types. They function just like the ones added through the editor, however the effects will just have the custom code I want to run within them.
